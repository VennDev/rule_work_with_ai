# Here are the principles based on my personal experience after 1–2 years of using AI in software development:

1. Clearly define the core idea and purpose of the software.
2. Identify the features and functionalities that make up the software.
3. Define the architecture, algorithms, and security considerations for the software.
4. Establish a logical and maintainable organization that fits the software's requirements.
5. Analyze and validate the feasibility of the idea before entering the testing phase. (This helps prevent situations where the project is nearly complete, only for you to discover a critical flaw in the original concept that forces you to redesign and rebuild everything from scratch.)
6. Ensure that the quality of test cases closely reflects real-world production scenarios.
7. Keep the AI agent's plugins and extensions limited to what is actually necessary for the project. Avoid excessive additions, as they provide little value and can unnecessarily bloat the AI's input context.
8. When bugs occur, make sure detailed debugging information is available so the root cause is clearly identified before asking AI to investigate or fix the issue. (I have encountered situations where insufficient debugging data forced the AI to make assumptions about the problem, leading to less effective troubleshooting and fixes.)
